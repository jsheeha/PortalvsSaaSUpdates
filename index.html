Shared Infrastructure

The Portal server is the underlying platform that all stores rely on. In Prod, each store is on its own server. Each server has the Portal server on it. We always make sure all stores are using the same version of Portal server.

Summary Comparison Table

While the "SaaS Update" target is more specific, due to integrations between stores, both update types require a full system maintenance window in prod due to the interconnected nature of the stores. This also means that integrations for all stores should be verified in testing.

The Integrated Reality

Even when only a SaaS store is being updated, all stores (including ASU's custom stores) must be taken offline.

  • Data Integrity: Prevents users from making changes in one store that cannot properly integrate with others during maintenance.
  • Validation: ASU custom stores (Grants, Agreements, Master) require integration testing after SaaS updates to ensure shared info remains accurate.

Manager’s Takeaway

SaaS Updates are store-specific code changes to the SaaS stores. Portal Updates apply to all stores. Both Update Types require testing the integrity of the ASU-maintained integrations for all stores and require production downtime for all stores simultaneously.
